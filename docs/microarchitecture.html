Section 01

Design Overview

This IP core implements a fully self-contained IEEE 1588 PTP Slave / Ordinary Clock synchronization engine for deployment in ProAV encoder or decoder SoCs connected to an Ethernet II network managed by a Mellanox Onyx Boundary Clock infrastructure. It operates as the slave endpoint of the Master–Slave timing hierarchy, recovering the Grandmaster clock with sub-100 nanosecond accuracy using hardware timestamping at the PHY layer.

Why Hardware PTP?

Software NTP provides only millisecond-range accuracy. IEEE 1588 PTP with hardware timestamping at the PHY/MAC boundary removes all OS-induced jitter from the timestamp path, enabling sub-microsecond, and with Boundary Clocks, sub-100 nanosecond accuracy across the LAN.

ProAV requirement: SMPTE ST 2059-2 mandates PTP-disciplined clocking for video/audio frame alignment. The GM hierarchy distributes a common reference derived from GPS/GNSS at domain 127.

Design Targets

Clock: 125 MHz system clock (8 ns resolution)
Accuracy goal: < 100 ns offset-from-master
PTP Domain: 127 (SMPTE ST 2059-2)
Profile: SMPTE ST 2059-2 / AES67 compatible
Transport: Ethernet II (EtherType 0x88F7)
Mode: Two-step slave-only
Section 02

Block Architecture

The IP is partitioned into three cooperating RTL submodules, all synchronous to a single 125 MHz clock domain. The wrapper (ptp_sync_wrapper) provides the single instantiation boundary for integration into the encoder/decoder SoC.

Section 03

Module Descriptions

ptp_sync_wrapper.sv

Integration Wrapper

Top-level integration boundary. Connects parser, sync core, and framer. Single module to instantiate in encoder/decoder SoC.

  • Wires AXI4-Stream RX/TX between MAC and submodules
  • Routes PHY hardware timestamps to appropriate sinks
  • Exposes all config and status to register file
ptp_slave_sync_top.sv

Sync Core

Central processing engine. Contains all PTP protocol logic, timestamp math, port FSM, BMCA, servo, and corrected clock.

  • Free-running 80-bit PTP hardware clock (48b sec + 32b ns)
  • Servo-adjusted corrected clock output
  • T1/T2/T3/T4 capture and OFM/MPD computation
  • BMCA with full 6-field comparison
  • 9-state port FSM per IEEE 1588-2008
  • PI servo with anti-windup (±10 ms clamp)
  • AMT / forced-master security checks
ptp_msg_parser.sv

Frame Parser

Byte-stream PTP frame decoder driven by AXI4-Stream from MAC RX. Extracts all IEEE 1588 header and body fields.

  • PTP over Ethernet_II (0x88F7) and IPv4/UDP
  • Hardware RX timestamp latch on SFD (tuser pulse)
  • Decode: Sync, Follow_Up, Delay_Resp, Announce
  • 44-byte header+body buffer with field extraction
ptp_delay_req_framer.sv

Delay_Req TX Framer

Builds and transmits Ethernet II Delay_Request PTP frames. Captures PHY TX hardware timestamp (T3) after SFD transmission.

  • 58-byte frame with SMPTE ST 2059-2 defaults
  • PTP multicast DA (01:1B:19:00:00:00)
  • 3-state FSM: IDLE → SEND → WAIT_TS
  • T3 precision from PHY HW timestamp feedback

Section 04

Timestamp Engine & Two-Step Mechanism

The timestamp engine implements the two-step synchronization exchange defined in Section 2 of the IEEE 1588 guide. Four hardware timestamps are captured per synchronization cycle and used to compute offset and delay.

T2 − T1 = Transmission_Delay + Clock_Offset (Forward path) T4 − T3 = Transmission_Delay − Clock_Offset (Return path) Clock_Offset = [ (T2 − T1) − (T4 − T3) ] / 2 Mean_Path_Dly = [ (T2 − T1) + (T4 − T3) ] / 2
Hardware timestamping: T2 is captured at the SFD boundary in the PHY timestamping unit (driven via rx_tuser strobe). T3 is the PHY TX hardware timestamp fed back after the Delay_Req SFD is transmitted (phy_tx_ts_valid strobe). This eliminates all software/OS-induced jitter from the measurement path.

Timestamp Format

All timestamps are 80-bit IEEE 1588 format: [79:32] = seconds (48-bit), [31:0] = nanoseconds (32-bit). The measurement engine converts these to a single 64-bit nanosecond value for arithmetic.

// 80-bit PTP timestamp → 64-bit nanoseconds function automatic logic signed [63:0] ts_to_ns ( input logic [79:0] ts); logic [47:0] secs = ts[79:32]; logic [31:0] nsec = ts[31:0]; return $signed({32'b0, secs[31:0]}) * 64'd1_000_000_000 + $signed({32'b0, nsec}); endfunction // Compute offset and mean path delay offset_calc <= ((t2_ns - t1_ns) - (t4_ns - t3_ns)) >>> 1; mpd_calc <= ((t2_ns - t1_ns) + (t4_ns - t3_ns)) >>> 1;
Section 05

PI Servo Controller

The servo controller is a fixed-point PI loop that adjusts the corrected clock's nanosecond register on each valid measurement. The correction is applied as an additive step (phase) to the corrected clock, simultaneously disciplining both phase and frequency over time via the integral term.

Controller Design

P-term = OFM >> Kp_shift (Kp = 2⁻⁴ = 1/16) I-term = integrator >> Ki_shift (Ki = 2⁻⁸ = 1/256) correction = −(P + I) integrator += OFM (anti-windup: ±10 ms)

Gains are fixed-point power-of-2 shifts to avoid multipliers. Kp=1/16 and Ki=1/256 are conservative values suitable for a low-jitter ProAV LAN with 8 Hz sync rate (SMPTE ST 2059-2 default).

Lock Detection

The clock_locked output is asserted when the measured offset-from-master falls within ±1 µs. This is a simplified single-sample criterion; production implementations should require N consecutive samples below threshold.

Anti-windup: Integrator is clamped to ±10,000,000 ns (±10 ms) to prevent runaway during long Announce-timeout periods or initial lock acquisition.

Clock Adjustment Path

// Apply servo step correction to corrected clock register if (servo_update && port_state == STATE_SLAVE) begin logic signed [NS_W:0] adjusted_ns; adjusted_ns = $signed({1'b0, corr_ns}) + servo_correction; if (adjusted_ns < 0) begin // Borrow from seconds corr_ns <= NS_PER_SEC + adjusted_ns[NS_W-1:0]; corr_sec <= corr_sec - 1'b1; end else if (adjusted_ns >= NS_PER_SEC) begin // Carry to seconds corr_ns <= adjusted_ns[NS_W-1:0] - NS_PER_SEC; corr_sec <= corr_sec + 1'b1; end else corr_ns <= adjusted_ns[NS_W-1:0]; end
Section 06

Best Master Clock Algorithm (BMCA)

The BMCA processes incoming Announce messages (after domain, AMT, and forced-master filtering) and selects the best master using a cascaded 6-field comparison per IEEE 1588-2008. The slave-only configuration prevents this device from ever propagating Announce messages.

Section 07

Port State Machine

The 9-state PTP port FSM governs when the IP participates in synchronization. For a slave-only ProAV device, the typical operational path is: INIT → LISTENING → UNCALIBRATED → SLAVE.

Section 08

Security Features

Acceptable Master Table (AMT)

An 8-entry ClockIdentity whitelist filters incoming Announce messages. Any Announce from a GM not in the table raises amt_violation and is silently discarded, preventing rogue time sources.

// Onyx equivalent: switch (config) # ptp amt <Clock ID> // RTL config port: cfg_amt_table[0] = 64'hAABB_CCDD_EEFF_0001; cfg_amt_valid[0] = 1'b1;

Forced-Master Port

When cfg_forced_master is asserted, all incoming Announce messages are discarded and logged via forced_master_event. This prevents directly-connected devices from influencing the BMCA or becoming an authoritative time source.

// Onyx equivalent: switch (config) # interface ethernet x/y \ ptp enable forced-master // RTL config port: cfg_forced_master = 1'b1;
Domain filter: Any PTP message with domainNumber ≠ cfg_domain_num (default 127 per SMPTE ST 2059-2) is silently discarded before any further processing. This prevents cross-domain interference in multi-tenant facilities.

Section 11

Verification Plan

TEST-01

Reset → LISTENING

Verify port state transitions from INIT to LISTENING within 3 cycles of reset deassertion.

TEST-02

BMCA Dataset Update

Inject Announce frames and verify GM identity and clock quality fields captured correctly in open-AMT mode.

TEST-03

AMT Violation

Load AMT with one entry, inject Announce from non-AMT GM, verify amt_violation asserted.

TEST-04

Forced-Master

Assert cfg_forced_master, inject Announce, verify forced_master_event and no BMCA update.

TEST-05

T1/T2 Capture

Inject Sync + Follow_Up frame pair with matching seqId. Verify T1 from preciseOriginTimestamp and T2 from PHY HW timestamp.

TEST-06

T3/T4 Exchange

Verify Delay_Req TX triggered at correct rate, T3 captured from phy_tx_ts_valid strobe, Delay_Resp seqId matching for T4.

TEST-07

OFM/MPD Computation

Inject known T1–T4 values, verify offset_from_master and mean_path_delay match analytical calculation within ±1 ns.

TEST-08

Servo Convergence

Apply sustained 500 ns OFM, verify corrected clock converges within 1 µs after N servo update cycles, clock_locked asserts.

TEST-09

Announce Timeout

Stop injecting Announce messages, verify announce_timer expires (3× interval), port state returns to LISTENING.

TEST-10

GM Failover

Switch Announce to new GM ClockIdentity, verify grandmaster_changed strobe, new GM ID captured, port re-enters UNCALIBRATED.

Simulation tool: The provided testbench targets any IEEE 1800-2012 compliant SystemVerilog simulator (Questa, Xcelium, VCS, Icarus Verilog 11+). Run: iverilog -g2012 -o sim *.sv && vvp sim or set up a Questa/VCS project including all five .sv files.
